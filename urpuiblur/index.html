Blur Algorithms
Six algorithms. One asset.

From the cheapest frosted-glass pass to full cinematic bokeh — pick the algorithm that fits your scene and budget. Switch at runtime with no pass rebuild.

DualKawase
Dual Kawase
Down/up mip pyramid — widest blur radius at the lowest cost. The default choice for frosted-glass UI panels on any platform.
CustomKawase
Custom Kawase
Circular ring blur with 3–12 tunable taps. CPU-precomputed directions — zero trig overhead per pixel.
PoissonDisk
Poisson Disk
Cinematic bokeh via Poisson disk sampling with optional IGN rotation. Eliminates grid artifacts even at heavy downsample rates.
GaussianSeparable
Gaussian Separable
Mathematically exact Gaussian circle in two separable passes. Visually perfect with zero ring artifacts.
TentHex
Tent / Hex
Three box-blur passes at 0°/60°/120° producing a hexagonal lens bokeh. The cinematic look for cutscenes and key moments.
Radial
Radial
Single-pass circular blur with per-pixel IGN rotation. Round bokeh feel — great for impact effects and transitions.
Performance
Built to run everywhere

Every part of the render pipeline is designed to do only what is needed, and nothing more. The result is a blur system that is fast by default and tunable for any platform budget.

Zero-cost consumers
Pass runs only when needed
A profile’s full pass chain stops running the moment its last active BlurImage is disabled or hidden. No consumers — no cost, automatically.
Frame throttling
Re-blur only as often as needed
Set updateMode to throttle blur updates per second. At 30 fps cap on a 180 fps game, two of every three frames reuse the cached result at zero added cost.
Freeze mode
Near-zero cost after first frame
Set throttleFps = 0 to blur once on activation and hold that frame indefinitely. Perfect for pause menus and modals over a still background.
Shared profiles
One pass chain, many panels
Multiple BlurImage components sharing the same BlurProfile pay the blur cost once. Extra panels are a material sample — not an extra pass.
Persistent RTHandles
No cross-profile aliasing
All intermediate textures are externally owned via RTHandles.Alloc(). The RenderGraph allocator cannot alias them — guaranteed isolation across all 16 active profiles.
Scissor culling
GPU work per-panel, not full-screen
Each blur pass is scissored to the panel’s screen bounds. Background pixels outside the panel are never shaded — cost scales with the panel’s visible area, not the screen.
Runtime API
Control blur from code

A set of focused APIs for the moments you need them — fading on menu open, driving intensity from an animation, throttling updates, or switching parameters at runtime.

FadeIn / FadeOut — coroutine-based, unscaled time 
// Animate intensity 0→1 over 0.4s. Works at Time.timeScale = 0.
// Reverses smoothly if called mid-animation.
blurImage.FadeIn(0.4f);
blurImage.FadeOut(0.3f);
SetIntensity / GetIntensity — manual intensity control 
// Drive intensity from any curve, tween library, or animation.
// Does not modify serialized parameter values.
myProfile.SetIntensity(0.5f);
float t = myProfile.GetIntensity();

// Or set directly
myProfile.intensity = Mathf.MoveTowards(myProfile.intensity, target, Time.deltaTime * 4f);
updateMode / throttleFps — frame throttling 
// Cap blur updates to 30/sec on a semi-static background.
myProfile.updateMode  = BlurUpdateMode.Throttled;
myProfile.throttleFps = 30;

// Freeze: blur once on open, hold for free until panel closes.
myProfile.throttleFps = 0;
BlurProfile fields — live parameter changes 
// All profile fields are writable at runtime.
// Changes take effect next frame -- no pass rebuild needed.
myProfile.blurOffset  = 2f;
myProfile.iterations  = 4;
myProfile.downsample  = 2;
myProfile.blurMethod  = BlurMethod.DualKawase;

Workflow
Working blur in under a minute

No shader graph wiring. No manual material setup. No URP pipeline boilerplate. The Setup Wizard handles the renderer, and the Inspector handles the rest.

01
Run the Setup Wizard
Opens automatically on first import. Scans every URP Renderer asset in the project and adds the Blur Renderer Feature to each one in a single click.
02
Add BlurImage to a UI panel
Add the component to any GameObject with an Image or RawImage. The material is created and managed automatically — no manual assignment needed.
03
Assign or create a BlurProfile
Click New in the Inspector to create a profile inline, or reuse an existing one across multiple panels. All blur parameters are live-editable in Play mode.
04
Press Play
The blur is active immediately. Tweak algorithm, radius, tint, and intensity live from the Inspector without stopping Play mode.
✎

Live Inspector editing

Every blur parameter, appearance field, and intensity slider updates in real time during Play mode. No recompile, no scene reload.

♠

Shared profiles

One BlurProfile asset can drive any number of panels. Change a parameter once and every panel using that profile updates together.
